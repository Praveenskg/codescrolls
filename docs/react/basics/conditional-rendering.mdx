# Conditional Rendering in React

**Conditional rendering** allows you to display different UI elements based on certain conditions. React provides several ways to conditionally render components.

---

## ğŸ“Œ Why Conditional Rendering?

You often need to show different content based on:
- User authentication status
- Loading states
- Error states
- User permissions
- Feature flags
- Data availability

---

## ğŸ”¸ If-Else Statements

The most basic way to conditionally render is using if-else statements:

```jsx
function Greeting({ isLoggedIn }) {
  if (isLoggedIn) {
    return <h1>Welcome back!</h1>;
  } else {
    return <h1>Please sign in.</h1>;
  }
}
```

### Early Return Pattern

```jsx
function UserDashboard({ user }) {
  // Early return for null/undefined
  if (!user) {
    return <div>Loading...</div>;
  }

  return (
    <div>
      <h1>Welcome, {user.name}!</h1>
      <p>Email: {user.email}</p>
    </div>
  );
}
```

## ğŸ”¸ Logical AND (&&) Operator

Use `&&` when you want to render something only if a condition is true:

```jsx
function Mailbox({ unreadMessages }) {
  return (
    <div>
      <h1>Hello!</h1>
      {unreadMessages.length > 0 && (
        <h2>You have {unreadMessages.length} unread messages.</h2>
      )}
    </div>
  );
}
```

### Multiple Conditions

```jsx
function UserProfile({ user, isAdmin }) {
  return (
    <div>
      <h1>{user.name}</h1>
      {user.email && <p>Email: {user.email}</p>}
      {user.phone && <p>Phone: {user.phone}</p>}
      {isAdmin && <button>Admin Panel</button>}
    </div>
  );
}
```

### âš ï¸ Beware of Falsy Values

```jsx
// âŒ Bad - Will render "0" if count is 0

// âœ… Good - Explicitly check
{count > 0 && <div>Count: {count}</div>}

// âœ… Also good
{count !== 0 && <div>Count: {count}</div>}
```

## ğŸ”¸ Logical OR (||) Operator

Use `||` to provide fallback content:

```jsx
function UserName({ name }) {
  return <h1>{name || 'Guest'}</h1>;
}
```

### Nullish Coalescing (??)

```jsx
function Counter({ count }) {
  // Only falls back if count is null or undefined (not 0)
  return <div>Count: {count ?? 'No count'}</div>;
}
```

## ğŸ”¸ Switch Statements

For multiple conditions, use switch statements:

```jsx

  switch (status) {
    case 'success':
      badge = <span className="badge-success">âœ“ Success</span>;
```

### Object Mapping (Cleaner Alternative)

```jsx

    success: <span className="badge-success">âœ“ Success</span>,
    error: <span className="badge-error">âœ— Error</span>,
```

## âœ… Best Practices

### 1. **Keep Conditions Simple**

```jsx
// âŒ Bad - Complex nested conditions
{isLoggedIn && user && user.isPremium && user.hasAccess && (
  <PremiumContent />
)}

// âœ… Good - Extract to variable
const canAccessPremium = isLoggedIn && user?.isPremium && user?.hasAccess;

{canAccessPremium && <PremiumContent />}
```

### 2. **Use Early Returns**

```jsx
// âŒ Bad - Nested conditions
function Component({ data }) {
  return (
    <div>
      {data ? (
        data.items ? (
          <List items={data.items} />
        ) : (
          <div>No items</div>
        )
      ) : (
        <div>Loading...</div>
      )}
    </div>
  );
}

// âœ… Good - Early returns
function Component({ data }) {
  if (!data) return <div>Loading...</div>;
  if (!data.items) return <div>No items</div>;
  
  return <List items={data.items} />;
}
```

### 3. **Extract Complex Conditions**

```jsx
// âŒ Bad
{user && user.age >= 18 && user.hasLicense && !user.isSuspended && (
  <DriveButton />
)}

// âœ… Good
const canDrive = user?.age >= 18 && user?.hasLicense && !user?.isSuspended;

{canDrive && <DriveButton />}
```

### 4. **Use Descriptive Variable Names**

```jsx
// âŒ Bad
const x = user && user.role === 'admin';

// âœ… Good
const isAdmin = user?.role === 'admin';
const canEditPost = isAdmin || post.authorId === user?.id;
```

## ğŸš¨ Common Mistakes

### 1. **Rendering Falsy Values**

```jsx
// âŒ Wrong - Renders "0"
{items.length && <div>Items: {items.length}</div>}

// âœ… Correct
{items.length > 0 && <div>Items: {items.length}</div>}
```

### 2. **Not Handling Null/Undefined**

```jsx
// âŒ Wrong - Can crash if user is null
{user.name && <div>{user.name}</div>}

// âœ… Correct - Use optional chaining
{user?.name && <div>{user.name}</div>}
```

### 3. **Too Many Ternaries**

```jsx
// âŒ Bad - Hard to read
{status === 'loading' ? <Spinner /> : status === 'error' ? <Error /> : status === 'success' ? <Success /> : <div>Unknown</div>}

// âœ… Good - Use if-else or object mapping
function StatusDisplay({ status }) {
  if (status === 'loading') return <Spinner />;
  if (status === 'error') return <Error />;
  if (status === 'success') return <Success />;
  return <div>Unknown</div>;
}
```

## ğŸ“ Summary

- Use **if-else** for complex logic with early returns
- Use **ternary operator** for inline conditional rendering
- Use **&&** for rendering when condition is true
- Use **||** or **??** for fallback values
- Use **switch** or **object mapping** for multiple conditions
- **Extract complex conditions** into variables
- **Handle null/undefined** with optional chaining
- **Avoid rendering falsy values** unintentionally
