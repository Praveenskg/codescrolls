# JSX in React â€“ JavaScript XML in React

JSX (JavaScript XML) is a **syntax extension used in React** that allows you to
write **HTML-like code inside JavaScript**. It makes React components easier to
read, write, and maintain.

If you are learning React, **understanding JSX is essential**, because almost
every React application uses JSX to describe the user interface.

---

## What Is JSX in React?

JSX stands for **JavaScript XML**. It lets developers describe UI elements using
a syntax that looks similar to HTML but works directly inside JavaScript code.

Although JSX looks like HTML, **it is not HTML**. JSX is converted into regular
JavaScript using `React.createElement()` before it runs in the browser.

---

## Why Use JSX?

JSX is popular in React because it improves both **developer experience** and
**code clarity**.

- Makes UI code **more readable**
- Keeps **markup and logic together**
- Allows **dynamic rendering** using JavaScript expressions
- Helps build **reusable components**

---

## Basic JSX Example

```jsx live
function Greeting() {
  return <h1>Hello, World!</h1>;
}
```

## JSX Rules You Must Know

- Return only **one root element** from a component
- Use `className` instead of `class`
- Event handlers like `onClick` are **camelCase**

```jsx live
function ButtonExample() {
  return <button onClick={() => alert('Clicked!')}>Click Me</button>;
}
```

## Convert HTML to JSX

Hereâ€™s how you convert a simple HTML snippet to JSX.

**HTML:**

```html
<div class="box">
  <h2>Title</h2>
  <p>Some text here.</p>
</div>
```

**JSX:**

```jsx live
function HtmlToJsx() {
  return (
    <div>
      <h2>Title</h2>
      <p>Some text here.</p>
    </div>
  );
}
```

---

## JSX Behind the Scenes

JSX is **syntactic sugar** for:

```jsx
<h1>Hello</h1>
```

which compiles to:

```js
React.createElement('h1', null, 'Hello');
```

---

## Recap

- JSX makes React code clean and readable
- You can embed JS expressions using `{}`
- JSX still compiles to `React.createElement` internally
