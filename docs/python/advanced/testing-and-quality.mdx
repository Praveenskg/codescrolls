# Python Testing and Code Quality

Quality workflows prevent regressions and simplify maintenance.

## Testing with pytest

```python
def add(a, b):
    return a + b

def test_add():
    assert add(2, 3) == 5
```

## Test Strategy

- Unit tests for core logic
- Integration tests for component boundaries
- Edge-case tests for failure paths

## Linting and Formatting

- Ruff or Flake8 for lint checks
- Black for formatting
- isort for import ordering

## CI Recommendation

Run tests, lint, and type checks on every pull request.
