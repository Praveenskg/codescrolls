# Python Variables and Data Types

Variables are labels that point to values in memory. Python is dynamically
typed, so you do not declare a variable type explicitly.

## Variable Basics

```python
name = "CodeScrolls"
age = 2
is_active = True
```

Python infers types at runtime.

## Naming Rules and Conventions

- Use `snake_case` for variable names (`total_price`)
- Start with a letter or underscore, not a digit
- Avoid Python keywords like `class`, `for`, `if`
- Use uppercase for constants by convention (`MAX_RETRIES = 3`)

## Core Data Types

- `str`: text values
- `int`: whole numbers
- `float`: decimal numbers
- `bool`: `True` or `False`
- `list`: ordered mutable collection
- `tuple`: ordered immutable collection
- `dict`: key-value mapping
- `set`: unordered unique values
- `NoneType`: absence of value (`None`)

## Type Checking

```python
value = 42
print(type(value))           # <class 'int'>
print(isinstance(value, int))  # True
```

Prefer `isinstance()` for checks in real code.

## Type Conversion

```python
num = int("10")
text = str(100)
price = float("19.99")
```

Be careful: invalid conversions raise `ValueError`.

## Mutable vs Immutable Types

- Mutable: `list`, `dict`, `set`
- Immutable: `int`, `float`, `str`, `tuple`, `bool`

```python
items = [1, 2]
items.append(3)  # list changes in place

text = "hi"
text = text + "!"  # creates a new string
```

## Common Pitfalls

- Reusing built-in names (`list`, `str`, `dict`) as variables
- Assuming input values are numbers (they are strings by default)
- Mutating shared objects unintentionally
