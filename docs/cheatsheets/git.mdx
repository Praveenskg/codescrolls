## ğŸ› ï¸ Fixing an Issue (Best Practice Workflow)

```bash
# 1. Make sure you're on the latest main branch
git checkout main
git pull origin main

# 2. Create a new branch for the fix
git checkout -b fix/bug-description

# 3. Make your code changes...

# 4. Stage and commit your changes
git add .
git commit -m "fix: meaningful message about the bug fix"

# 5. Push the branch to origin
git push origin fix/bug-description

# 6. Open a Pull Request on GitHub
# (Describe the issue and how it was fixed)

# 7. After review, merge into main
# (Use GitHub UI or CLI)

# 8. Delete local & remote branch
git branch -d fix/bug-description
git push origin --delete fix/bug-description
```

- Use conventional commit messages like `fix:` or `bug:`.
- Always pull the latest changes before starting.
