# Git Cheatsheet

A handy reference for common Git commands used in day-to-day development.

---

![Git Workflow](/img/skills/Git.png)

## Setup

```bash
# Set your identity
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

# Check your configuration
git config --list
```

---

## Start a Project

```bash
# Initialize a new repository
git init

# Clone an existing repository
git clone https://github.com/user/repo.git
```

---

## Working with Files

```bash
# Check status of changes
git status

# Stage files
git add filename
git add .  # stage all changes

# Commit changes
git commit -m "Your message"

# Remove a file from the repo
git rm filename
```

---

## Branching

```bash
# List branches
git branch

# Create a new branch
git branch new-feature

# Switch to a branch (modern way)
git switch new-feature

# Switch to a branch (legacy way, still works)
git checkout new-feature

# Create and switch in one step (modern way)
git switch -c new-feature

# Create and switch in one step (legacy way)
git checkout -b new-feature
```

---

## Merging and Rebase

```bash
# Merge a branch into the current one
git merge branch-name

# Rebase current branch onto another
git rebase main
```

---

## Push and Pull

```bash
# Push to remote and set upstream (recommended)
git push -u origin branch-name

# Push to remote (after upstream is set)
git push

# Push to remote (explicit)
git push origin branch-name

# Pull from remote with rebase (recommended)
git pull --rebase origin branch-name

# Pull from remote (merge strategy)
git pull origin branch-name
```

---

## Clean and Undo

```bash
# Discard local changes (modern way)
git restore filename

# Discard local changes (legacy way)
git checkout -- filename

# Unstage a file (modern way)
git restore --staged filename

# Unstage a file (legacy way)
git reset filename

# Undo last commit but keep changes
git reset --soft HEAD~1
```

## Logs & History

```bash
# View commit log
git log

# One-line summary
git log --oneline

# Show file history with author info
git blame filename
```

## Fixing an Issue (Recommended Workflow)

```bash
# 1. Update your main branch
git switch main
git pull --rebase origin main

# 2. Create a new branch for the fix
git switch -c fix/bug-description  # or: git checkout -b fix/bug-description

# 3. Make your changes...

# 4. Stage and commit
git add .
git commit -m "fix: describe the bug fix"

# 5. Push the branch to origin and set upstream
git push -u origin fix/bug-description

# 6. Open a Pull Request on GitHub

# 7. After review, merge into main

# 8. Delete local & remote branch
git branch -d fix/bug-description
git push origin --delete fix/bug-description
```
:::tip  Tips
- Use clear and conventional commit messages like `fix:`, `feat:`, or `chore:`.
- Always pull the latest changes before creating new branches.
- Keep commits small and focused for easier reviews.
- Use `.gitignore` to avoid committing unnecessary files.
- Use `git stash` to temporarily save work in progress.
- Prefer SSH over HTTPS for authentication.
:::

## Advanced Git

### Cherry-pick

```bash
# Apply a specific commit from another branch
git cherry-pick <commit-hash>
```

### Revert

```bash
# Create a new commit that undoes a previous commit
git revert <commit-hash>
```

### Reflog

```bash
# Show history of HEAD changes (including resets & checkouts)
git reflog

# Restore a deleted branch/commit (modern way)
git switch -c recovered-branch <commit-hash>

# Restore a deleted branch/commit (legacy way)
git checkout <commit-hash>
```

### Reset (Hard vs Soft)

```bash
# Move HEAD to a previous commit, keep changes
git reset --soft <commit-hash>

# Move HEAD to a previous commit, discard changes
git reset --hard <commit-hash>
```

### Bisect

```bash
# Start binary search to find a bad commit
git bisect start
git bisect bad          # mark current commit as bad
git bisect good <hash>  # mark a known good commit
```

### Stash

```bash
# Save current changes temporarily
git stash

# List stashes
git stash list

# Apply the most recent stash
git stash apply

# Apply and drop stash
git stash pop
```

### Tags & Releases

```bash
# Create a lightweight tag
git tag v1.0.0

# Create an annotated tag
git tag -a v1.0.0 -m "Release v1.0.0"

# Push tags to remote (all tags)
git push origin --tags

# Push tags with commits (recommended - only pushes tags for commits being pushed)
git push --follow-tags
```
### Manual Release Workflow (Git + npm)

```bash
# 1. Bump version (patch / minor / major)
npm version patch   # or minor / major

# 2. Create a Git tag (auto created by npm version also)
git tag v1.0.0 -m "Release v1.0.0"

# 3. Push commits and tags to remote (recommended)
git push --follow-tags origin main

# Alternative: Push separately
git push origin main
git push origin --tags

# 4. Publish to npm (make sure you are logged in: npm login)
npm publish --access public

# 5. (Optional) Create GitHub Release with changelog
gh release create v1.0.0 --title "Release v1.0.0" --notes "Changelog details here"
```
:::tip Notes

Use `npm version patch` (or minor/major) instead of manually editing package.json version.

This ensures commit + tag are created together.

Always `run npm test && npm run build` before publishing.

If using 2FA on npm, set it to Authorization only for CI/CD.
:::

### Submodules

```bash
# Add a submodule
git submodule add https://github.com/user/repo.git path/to/repo

# Initialize and fetch submodules
git submodule update --init --recursive

# Update submodules
git submodule update --remote --merge
```

## Troubleshooting Common Issues

### Merge Conflicts

```bash
# See conflicting files
git status

# Edit the files to resolve conflicts, then stage them
git add filename

# Continue merge/rebase after resolving
git merge --continue
git rebase --continue
```

### Detached HEAD

```bash
# You are on a commit, not a branch
# To return to a branch (modern way)
git switch main

# To return to a branch (legacy way)
git checkout main

# Or create a new branch from this commit (modern way)
git switch -c new-branch

# Or create a new branch from this commit (legacy way)
git checkout -b new-branch
```

### Push Rejected (Non-Fast-Forward)

```bash
# Pull latest changes before pushing
git pull --rebase origin main

# Then push
git push origin main
```

### Accidentally Deleted a Branch

```bash
# Find commit in reflog
git reflog

# Restore the branch (modern way)
git switch -c branch-name <commit-hash>

# Restore the branch (legacy way)
git checkout -b branch-name <commit-hash>
```

### Overwritten Local Changes

```bash
# Recover changes if lost
git reflog

# Check out a previous commit with your changes (modern way)
git switch -c recovered-branch <commit-hash>

# Check out a previous commit with your changes (legacy way)
git checkout <commit-hash>
```

### Undo a Merge/Rebase

```bash
# Abort a merge
git merge --abort

# Abort a rebase
git rebase --abort
```

## Git Hooks

### Pre-commit Hook

```bash
# Create pre-commit hook
echo '#!/bin/sh
npm test
npm run lint
' > .git/hooks/pre-commit

chmod +x .git/hooks/pre-commit
```

### Commit Message Hook

```bash
# Create commit-msg hook for conventional commits
echo '#!/bin/sh
commit_regex="^(feat|fix|docs|style|refactor|perf|test|build|chore)(\(.+\))?: .{1,50}"
if ! grep -qE "$commit_regex" "$1"; then
    echo "Invalid commit message format!"
    echo "Use: type(scope): description"
    echo "Types: feat, fix, docs, style, refactor, perf, test, build, chore"
    exit 1
fi
' > .git/hooks/commit-msg

chmod +x .git/hooks/commit-msg
```

## Git Aliases

### Useful Aliases

```bash
# Add to ~/.gitconfig
[alias]
    st = status
    co = checkout
    br = branch
    ci = commit
    unstage = reset HEAD --
    last = log -1 HEAD
    visual = !gitk
    lg = log --oneline --graph --decorate --all
    amend = commit --amend --no-edit
    undo = reset HEAD~1 --mixed
    wip = commit -am "WIP"
    unwip = reset HEAD~1
```

### Advanced Aliases

```bash
[alias]
    # Show file changes in last commit
    lastdiff = diff HEAD~1 HEAD
    
    # Show commits by author
    author = !git log --author="$1" --oneline
    
    # Show files changed in last commit
    lastfiles = diff --name-only HEAD~1 HEAD
    
    # Interactive rebase for last N commits
    rebasei = "!f() { git rebase -i HEAD~$1; }; f"
    
    # Create and switch to new branch (modern way)
    new = "!f() { git switch -c $1; }; f"
    
    # Create and switch to new branch (legacy way)
    # new = "!f() { git checkout -b $1; }; f"
    
    # Delete merged branches
    cleanup = "!git branch --merged | grep -v '\\*\\|main\\|develop' | xargs -n 1 git branch -d"
```

## Git Workflows

### Git Flow

```bash
# Initialize git flow
git flow init

# Start a feature
git flow feature start new-feature

# Finish a feature
git flow feature finish new-feature

# Start a release
git flow release start 1.0.0

# Finish a release
git flow release finish 1.0.0
```

### GitHub Flow

```bash
# 1. Create feature branch from main
git switch main
git pull --rebase origin main
git switch -c feature/new-feature

# 2. Make changes and commit
git add .
git commit -m "feat: add new feature"

# 3. Push and create PR
git push -u origin feature/new-feature

# 4. After PR merge, clean up
git switch main
git pull --rebase origin main
git branch -d feature/new-feature
```

### GitLab Flow

```bash
# 1. Create branch from main
git switch main
git pull --rebase origin main
git switch -c feature/new-feature

# 2. Work on feature
git add .
git commit -m "feat: implement new feature"

# 3. Push to feature branch
git push -u origin feature/new-feature

# 4. Create merge request
# 5. After merge, delete branch
git branch -d feature/new-feature
```

## Advanced Git Commands

### Interactive Rebase

```bash
# Rebase last 3 commits
git rebase -i HEAD~3

# During interactive rebase:
# pick - use commit
# reword - use commit, but edit message
# edit - use commit, but stop for amending
# squash - use commit, but meld into previous
# drop - remove commit
```

### Partial Staging

```bash
# Stage parts of a file
git add -p filename

# Stage specific lines
git add -p

# During patch mode:
# y - stage this hunk
# n - don't stage this hunk
# s - split into smaller hunks
# e - manually edit the hunk
```

### Advanced Logging

```bash
# Show commits with file changes
git log --stat

# Show commits with patches
git log -p

# Show commits in a specific date range
git log --since="2024-01-01" --until="2024-12-31"

# Show commits by author
git log --author="John Doe"

# Show commits affecting specific files
git log -- filename1 filename2

# Show commits with graph
git log --oneline --graph --decorate --all

# Show commits with custom format
git log --pretty=format:"%h - %an, %ar : %s"
```

### Advanced Diff

```bash
# Show diff between branches
git diff branch1..branch2

# Show diff between commits
git diff commit1..commit2

# Show diff for specific file
git diff HEAD~1 HEAD -- filename

# Show diff with word-level changes
git diff --word-diff

# Show diff with color
git diff --color-words

# Show diff ignoring whitespace
git diff -w
```

### Advanced Branching

```bash
# List all branches (local and remote)
git branch -a

# List remote branches
git branch -r

# Create branch from specific commit (modern way)
git switch -c new-branch <commit-hash>

# Create branch from specific commit (legacy way)
git checkout -b new-branch <commit-hash>

# Create branch from remote branch (modern way)
git switch -c local-branch origin/remote-branch

# Create branch from remote branch (legacy way)
git checkout -b local-branch origin/remote-branch

# Set upstream for existing branch
git branch --set-upstream-to=origin/main main

# Rename branch
git branch -m old-name new-name

# Delete remote branch
git push origin --delete branch-name
```

### Advanced Merging

```bash
# Merge with no fast-forward
git merge --no-ff feature-branch

# Merge with squash
git merge --squash feature-branch

# Merge specific files from another branch (modern way)
git restore --source=feature-branch -- file1 file2

# Merge specific files from another branch (legacy way)
git checkout feature-branch -- file1 file2

# Merge with custom merge tool
git merge --tool=vimdiff feature-branch
```

### Advanced Stashing

```bash
# Stash with message
git stash push -m "Work in progress on feature X"

# Stash specific files
git stash push -m "Stash specific files" file1 file2

# Stash including untracked files
git stash push -u

# Stash including ignored files
git stash push -a

# Apply specific stash
git stash apply stash@{2}

# Show stash contents
git stash show -p stash@{0}

# Create branch from stash
git stash branch new-branch stash@{0}
```

## Git Configuration

### Global Configuration

```bash
# Set default branch name
git config --global init.defaultBranch main

# Set default editor
git config --global core.editor "code --wait"

# Set default merge tool
git config --global merge.tool vscode

# Set line ending handling
git config --global core.autocrlf input  # Linux/Mac
git config --global core.autocrlf true   # Windows

# Set credential helper (platform-specific, more secure)
git config --global credential.helper cache  # Linux/Mac (temporary, 15 min default)
git config --global credential.helper manager  # Windows (Git Credential Manager)
git config --global credential.helper osxkeychain  # macOS (Keychain)

# Store credentials permanently (less secure, not recommended)
# git config --global credential.helper store

# Set push default
git config --global push.default simple

# Set pull strategy
git config --global pull.rebase false
```

### Repository-specific Configuration

```bash
# Set local user for this repo
git config user.name "Project Name"
git config user.email "project@example.com"

# Set local branch protection
git config branch.main.protect true

# Set local merge strategy
git config merge.ours.driver true
```

## Git Best Practices

### Commit Message Guidelines

```bash
# Conventional Commits Format
<type>[optional scope]: <description>

[optional body]

[optional footer(s)]

# Types:
feat:     A new feature
fix:      A bug fix
docs:     Documentation only changes
style:    Changes that do not affect the meaning of the code
refactor: A code change that neither fixes a bug nor adds a feature
perf:     A code change that improves performance
test:     Adding missing tests or correcting existing tests
build:    Changes that affect the build system or external dependencies
chore:    Changes to the build process or auxiliary tools
```

### Branch Naming Conventions

```bash
# Feature branches
feature/user-authentication
feature/payment-integration

# Bug fix branches
fix/login-error
fix/memory-leak

# Hotfix branches
hotfix/security-patch
hotfix/critical-bug

# Release branches
release/v1.2.0
release/v2.0.0

# Chore branches
chore/update-dependencies
chore/refactor-components
```

### File Organization

```bash
# .gitignore best practices
# Dependencies

vendor/

# Build outputs

build/
*.min.js
*.min.css

# Environment files
.env
.env.local
.env.production

# IDE files
.vscode/
.idea/
*.swp
*.swo

# OS files
.DS_Store
Thumbs.db

# Logs
*.log

pids/

```

## Git Security

### SSH Key Setup

```bash
# Generate SSH key
ssh-keygen -t ed25519 -C "your_email@example.com"

# Add to SSH agent
ssh-add ~/.ssh/id_ed25519

# Copy public key
cat ~/.ssh/id_ed25519.pub

# Test SSH connection
ssh -T git@github.com
```

### GPG Signing

```bash
# Generate GPG key
gpg --full-generate-key

# List GPG keys
gpg --list-secret-keys --keyid-format LONG

# Configure Git to use GPG
git config --global user.signingkey <GPG_KEY_ID>
git config --global commit.gpgsign true

# Sign commits
git commit -S -m "Signed commit message"
```
