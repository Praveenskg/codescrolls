# Git Cheatsheet

A handy reference for common Git commands used in day-to-day development.

---

![Git Workflow](/img/skills/Git.png)

## Setup

```bash
# Set your identity
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

# Check your configuration
git config --list
```

---

## Start a Project

```bash
# Initialize a new repository
git init

# Clone an existing repository
git clone https://github.com/user/repo.git
```

---

## Working with Files

```bash
# Check status of changes
git status

# Stage files
git add filename
git add .  # stage all changes

# Commit changes
git commit -m "Your message"

# Remove a file from the repo
git rm filename
```

---

## Branching

```bash
# List branches
git branch

# Create a new branch
git branch new-feature

# Switch to a branch
git checkout new-feature

# Create and switch in one step
git checkout -b new-feature
```

---

## Merging and Rebase

```bash
# Merge a branch into the current one
git merge branch-name

# Rebase current branch onto another
git rebase main
```

---

## Push and Pull

```bash
# Push to remote
git push origin branch-name

# Pull from remote
git pull origin branch-name
```

---

## Clean and Undo

```bash
# Discard local changes
git checkout -- filename

# Unstage a file
git reset filename

# Undo last commit but keep changes
git reset --soft HEAD~1
```

---

## Logs & History

```bash
# View commit log
git log

# One-line summary
git log --oneline

# Show file history with author info
git blame filename
```

---

## Fixing an Issue (Recommended Workflow)

```bash
# 1. Update your main branch
git checkout main
git pull origin main

# 2. Create a new branch for the fix
git checkout -b fix/bug-description

# 3. Make your changes...

# 4. Stage and commit
git add .
git commit -m "fix: describe the bug fix"

# 5. Push the branch to origin
git push origin fix/bug-description

# 6. Open a Pull Request on GitHub

# 7. After review, merge into main

# 8. Delete local & remote branch
git branch -d fix/bug-description
git push origin --delete fix/bug-description
```
:::tip  Tips
- Use clear and conventional commit messages like `fix:`, `feat:`, or `chore:`.
- Always pull the latest changes before creating new branches.
- Keep commits small and focused for easier reviews.
- Use `.gitignore` to avoid committing unnecessary files.
- Use `git stash` to temporarily save work in progress.
- Prefer SSH over HTTPS for authentication.
:::

## Advanced Git

### Cherry-pick

```bash
# Apply a specific commit from another branch
git cherry-pick <commit-hash>
```

### Revert

```bash
# Create a new commit that undoes a previous commit
git revert <commit-hash>
```

### Reflog

```bash
# Show history of HEAD changes (including resets & checkouts)
git reflog

# Restore a deleted branch/commit
git checkout <commit-hash>
```

### Reset (Hard vs Soft)

```bash
# Move HEAD to a previous commit, keep changes
git reset --soft <commit-hash>

# Move HEAD to a previous commit, discard changes
git reset --hard <commit-hash>
```

### Bisect

```bash
# Start binary search to find a bad commit
git bisect start
git bisect bad          # mark current commit as bad
git bisect good <hash>  # mark a known good commit
```

### Stash

```bash
# Save current changes temporarily
git stash

# List stashes
git stash list

# Apply the most recent stash
git stash apply

# Apply and drop stash
git stash pop
```

### Tags & Releases

```bash
# Create a lightweight tag
git tag v1.0.0

# Create an annotated tag
git tag -a v1.0.0 -m "Release v1.0.0"

# Push tags to remote
git push origin --tags
```
### Manual Release Workflow (Git + npm)

```bash
# 1. Bump version (patch / minor / major)
npm version patch   # or minor / major

# 2. Create a Git tag (auto created by npm version also)
git tag v1.0.0 -m "Release v1.0.0"

# 3. Push commits and tags to remote
git push origin main
git push origin --tags

# 4. Publish to npm (make sure you are logged in: npm login)
npm publish --access public

# 5. (Optional) Create GitHub Release with changelog
gh release create v1.0.0 --title "Release v1.0.0" --notes "Changelog details here"
```
:::tip Notes

Use `npm version patch` (or minor/major) instead of manually editing package.json version.

This ensures commit + tag are created together.

Always `run npm test && npm run build` before publishing.

If using 2FA on npm, set it to Authorization only for CI/CD.
:::

### Submodules

```bash
# Add a submodule
git submodule add https://github.com/user/repo.git path/to/repo

# Initialize and fetch submodules
git submodule update --init --recursive

# Update submodules
git submodule update --remote --merge
```

## Troubleshooting Common Issues

### Merge Conflicts

```bash
# See conflicting files
git status

# Edit the files to resolve conflicts, then stage them
git add filename

# Continue merge/rebase after resolving
git merge --continue
git rebase --continue
```

### Detached HEAD

```bash
# You are on a commit, not a branch
# To return to a branch
git checkout main

# Or create a new branch from this commit
git checkout -b new-branch
```

### Push Rejected (Non-Fast-Forward)

```bash
# Pull latest changes before pushing
git pull --rebase origin main

# Then push
git push origin main
```

### Accidentally Deleted a Branch

```bash
# Find commit in reflog
git reflog

# Restore the branch
git checkout -b branch-name <commit-hash>
```

### Overwritten Local Changes

```bash
# Recover changes if lost
git reflog

# Check out a previous commit with your changes
git checkout <commit-hash>
```

### Undo a Merge/Rebase

```bash
# Abort a merge
git merge --abort

# Abort a rebase
git rebase --abort
```

## GitHub CLI Workflow

### Authentication

```bash
# Login to GitHub
gh auth login

# Check authentication status
gh auth status
```

### Pull Requests

```bash
# Create a new pull request
gh pr create --title "fix: bug description" --body "Details about the fix"

# List open PRs
gh pr list

# Checkout a pull request locally
gh pr checkout <pr-number>

# Merge a pull request
gh pr merge <pr-number>
```

### Issues

```bash
# Create a new issue
gh issue create --title "Bug: something broke" --body "Steps to reproduce..."

# List issues
gh issue list

# View details of an issue
gh issue view <issue-number>
```

### Releases

```bash
# Create a new release
gh release create v1.0.0 --title "Release v1.0.0" --notes "Changelog details here"

# List releases
gh release list

# View release details
gh release view v1.0.0
```

### Gists

```bash
# Create a gist from a file
gh gist create file.txt

# List your gists
gh gist list
```
