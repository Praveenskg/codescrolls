## üåê HTTP Methods

### **Core HTTP Methods**

```http
GET     /users          # Retrieve data (safe, idempotent)
POST    /users          # Create new resource (not idempotent)
PUT     /users/123      # Update entire resource (idempotent)
PATCH   /users/123      # Partial update (not idempotent)
DELETE  /users/123      # Remove resource (idempotent)
HEAD    /users          # Get headers only (safe, idempotent)
OPTIONS /users          # Get allowed methods (safe, idempotent)
```

```http

```

```http

203 Non-Authoritative Information # Response from cache

206 Partial Content   # Partial response

226 IM Used          # Instance manipulation used
```

## üîê Authentication Methods

### **API Key Authentication**

```http
# Query Parameter
GET /api/users?api_key=your_api_key_here

# Header
X-API-Key: your_api_key_here
Authorization: ApiKey your_api_key_here

# Basic Auth (API Key as username)
Authorization: Basic base64(api_key:)
```

### **Bearer Token (JWT)**

```http
# Most common for modern APIs
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

# Compact format
Authorization: Bearer abc123def456
```

### **Basic Authentication**

```http
# Username:Password encoded in Base64
Authorization: Basic dXNlcjpwYXNz

# Example: user="admin", pass="secret"
# Base64: YWRtaW46c2VjcmV0
Authorization: Basic YWRtaW46c2VjcmV0
```

### **OAuth 2.0 Flow**

```http
# 1. Authorization Request
GET /oauth/authorize?response_type=code&client_id=123&redirect_uri=https://app.com/callback&scope=read+write

# 2. Authorization Code
HTTP/1.1 302 Found
Location: https://app.com/callback?code=abc123&state=xyz

# 3. Token Exchange
POST /oauth/token
Content-Type: application/x-www-form-urlencoded

grant_type=authorization_code&code=abc123&redirect_uri=https://app.com/callback&client_id=123&client_secret=secret

# 4. Token Response
{
  "access_token": "abc123",
  "token_type": "Bearer",
  "expires_in": 3600,
  "refresh_token": "refresh123"
}
```

### **Digest Authentication**

```http
# Challenge from server
WWW-Authenticate: Digest realm="api@example.com", nonce="abc123", algorithm=MD5, qop="auth"

# Client response
Authorization: Digest username="user", realm="api@example.com", nonce="abc123", uri="/api/users", response="hash...", qop=auth, nc=00000001, cnonce="def456"
```

## üìù API Design Patterns

### **RESTful Resource Naming**

#### **Basic Resources**

```http
GET    /users           # List users
GET    /users/123       # Get specific user
POST   /users           # Create user
PUT    /users/123       # Update user (full)
PATCH  /users/123       # Update user (partial)
DELETE /users/123       # Delete user
```

#### **Nested Resources**

```http
GET    /users/123/posts       # User's posts
GET    /users/123/posts/456   # Specific post
POST   /users/123/posts       # Create post for user
PUT    /posts/456             # Update post (direct)
DELETE /users/123/posts/456   # Delete user's post
```

#### **Actions on Resources**

```http
POST   /users/123/activate    # Action endpoints
POST   /users/123/deactivate
POST   /users/123/reset-password
GET    /users/123/avatar      # Resource sub-components
PUT    /users/123/avatar
```

### **Query Parameters**

#### **Pagination**

```http
GET /users?page=1&limit=10&sort=name&order=asc
GET /users?offset=20&limit=10
GET /users?cursor=eyJpZCI6MTIzfQ==&limit=10
```

#### **Filtering**

```http
GET /users?status=active
GET /users?age[gte]=18&age[lte]=65
GET /users?role=admin,moderator
GET /products?category=electronics&price_min=100&price_max=500
GET /posts?published=true&author=john&tags=javascript,react
```

#### **Searching & Sorting**

```http
GET /users?search=john&fields=name,email,role
GET /products?q=laptop&sort=price&order=desc
GET /articles?tags=javascript&sort=published_at&order=desc
```

#### **Field Selection**

```http
GET /users?fields=id,name,email,created_at
GET /users?include=posts,comments
GET /users?exclude=password,deleted_at
```

### **Content Negotiation**

#### **Accept Header**

```http
# Request specific format
GET /users/123
Accept: application/json

GET /users/123
Accept: application/xml

GET /users/123
Accept: text/csv
```

#### **Content-Type Header**

```http
# Specify request body format
POST /users
Content-Type: application/json

{
  "name": "John Doe",
  "email": "john@example.com"
}
```

### **cURL Commands**

```bash
# GET request
curl -X GET "https://api.example.com/users" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Accept: application/json"

# POST request
curl -X POST "https://api.example.com/users" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"John","email":"john@example.com"}'

# PUT request
curl -X PUT "https://api.example.com/users/123" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"John Doe","email":"john@example.com"}'

# DELETE request
curl -X DELETE "https://api.example.com/users/123" \
  -H "Authorization: Bearer YOUR_TOKEN"

# Upload file
curl -X POST "https://api.example.com/upload" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@/path/to/file.jpg" \
  -F "description=Profile picture"

# Verbose output
curl -v -X GET "https://api.example.com/users"
```

## ‚ö° Performance & Caching

### **HTTP Caching Headers**

```http
# Cache-Control directives
Cache-Control: no-cache              # Revalidate before using
Cache-Control: no-store              # Don't cache
Cache-Control: max-age=3600          # Cache for 1 hour
Cache-Control: private               # User-specific cache
Cache-Control: public                # Shared cache allowed

# ETags for conditional requests
ETag: "33a64df551425fcc55e4d42a148795d9f25f89d4"
If-None-Match: "33a64df551425fcc55e4d42a148795d9f25f89d4"

# Last-Modified for conditional requests
Last-Modified: Wed, 21 Oct 2023 07:28:00 GMT
If-Modified-Since: Wed, 21 Oct 2023 07:28:00 GMT
```

### **Rate Limiting Headers**

```http
# Rate limit information
X-RateLimit-Limit: 1000         # Max requests per hour
X-RateLimit-Remaining: 950      # Remaining requests
X-RateLimit-Reset: 1640995200   # Unix timestamp when limit resets
X-RateLimit-Retry-After: 3600   # Seconds to wait

# Retry-After for 429 responses
Retry-After: 3600               # Seconds
Retry-After: Wed, 21 Oct 2023 08:28:00 GMT  # Date
```

### **Compression**

```http
# Request compression
Accept-Encoding: gzip, deflate, br

# Response compression
Content-Encoding: gzip
Content-Length: 348

# Vary header for content negotiation
Vary: Accept-Encoding
```

### **Input Validation**

```json
// Whitelist validation
{
  "name": "John Doe",           // Required, string, max 100 chars
  "email": "john@example.com",  // Required, valid email format
  "age": 25,                    // Optional, integer, 13-120
  "role": "user"                // Optional, enum: admin|moderator|user

// Sanitization examples
{
  "comment": "<script>alert('xss')</script>",  // Becomes: alert('xss')
  "sql": "'; DROP TABLE users; --",           // Properly escaped
  "path": "../../../etc/passwd"               // Normalized to safe path
}
```


```http

‚ùå GET /getUserPosts/123

# 2. Use HTTP methods correctly

‚úÖ GET /users
‚ùå GET /user

# 4. Use sub-resources for relationships
‚úÖ GET /users/123/posts
‚ùå GET /posts?user_id=123

# 5. Handle trailing slashes consistently
```
