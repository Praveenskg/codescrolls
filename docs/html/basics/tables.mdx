# üìä HTML Tables

HTML tables organize and display data in rows and columns. Modern tables include
accessibility features, responsive design, and semantic structure.

## üìñ Table Basics

### Basic Table Structure

```html
<table>
  <thead>
    <tr>
      <th>Name</th>
      <th>Age</th>
      <th>City</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>John Doe</td>
      <td>25</td>
      <td>New York</td>
    </tr>
    <tr>
      <td>Jane Smith</td>
      <td>30</td>
      <td>London</td>
    </tr>
  </tbody>
</table>
```

### Table Elements

```html
<table>
  <!-- Table caption (accessibility) -->
  <caption>
    Employee Information
  </caption>

  <!-- Table header -->
  <thead>
    <tr>
      <th>Name</th>
      <th>Position</th>
      <th>Salary</th>
    </tr>
  </thead>

  <!-- Table body -->
  <tbody>
    <tr>
      <td>John Doe</td>
      <td>Developer</td>
      <td>$75,000</td>
    </tr>
    <tr>
      <td>Jane Smith</td>
      <td>Designer</td>
      <td>$70,000</td>
    </tr>
  </tbody>

  <!-- Table footer -->
  <tfoot>
    <tr>
      <th colspan="2">Total Employees</th>
      <td>2</td>
    </tr>
  </tfoot>
</table>
```

### Header Types

```html
<table>
  <thead>
    <tr>
      <th>Regular Header</th>
      <th scope="col">Column Header</th>

  <tbody>
    <th scope="row">Q1</th>

    <td>$10,000</td>
```

## ‚ôø Accessible Tables

### Proper Headers

```html
<table>
  <caption>
    Employee Performance Review
  </caption>
  <thead>
    <tr>
      <th id="name">Employee Name</th>
      <th id="rating">Performance Rating</th>
      <th id="comments">Comments</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th headers="name">John Doe</th>
      <td headers="rating">Excellent</td>
      <td headers="comments">Consistently exceeds expectations</td>
    </tr>
  </tbody>
</table>
```

### Table Summary

```html
<table aria-describedby="table-summary">
  <caption id="table-summary">
    This table shows quarterly sales data for three products across four
    regions. Data is in thousands of dollars.
  </caption>
  <!-- Table content -->
</table>
```

### Landmark Tables

```html
<main>
  <h1>Sales Report</h1>

  <table role="table" aria-label="Quarterly sales by product">
    <!-- Table content -->
  </table>
</main>
```

## üì± Responsive Tables

### Mobile-First Approach

```html
<style>
  .responsive-table {
    width: 100%;
    border-collapse: collapse;
  }

  .responsive-table th,
  .responsive-table td {
    padding: 8px 12px;
    text-align: left;
  }

  /* Hide table headers on mobile */
  @media screen and (max-width: 768px) {
    .responsive-table thead {
      display: none;
    }

    .responsive-table tr {
      display: block;
      border: 1px solid #ddd;
      margin-bottom: 10px;
      border-radius: 4px;
    }

    .responsive-table td {
      display: block;
      text-align: right;
      border: none;
      border-bottom: 1px solid #eee;
      position: relative;
      padding-left: 50%;
    }

    .responsive-table td:before {
      content: attr(data-label);
      position: absolute;
      left: 12px;
      font-weight: bold;
      text-transform: uppercase;
      font-size: 0.8em;
      color: #666;
    }

    .responsive-table td:last-child {
      border-bottom: none;
    }
  }
</style>

<table class="responsive-table">
  <thead>
    <tr>
      <th>Name</th>
      <th>Email</th>
      <th>Phone</th>
      <th>Department</th>
    </tr>
  </thead>

    <tr>
      <td data-label="Name">John Doe</td>
      <td data-label="Email">john@example.com</td>
```

## üìä Data Tables

### Sortable Tables

```html
<table class="sortable-table">
  <thead>
    <tr>
      <th data-sort="string">Name</th>
      <th data-sort="number">Age</th>
      <th data-sort="string">City</th>
      <th data-sort="date">Join Date</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>John Doe</td>
      <td>25</td>
      <td>New York</td>
      <td>2023-01-15</td>
    </tr>
    <tr>
      <td>Jane Smith</td>
      <td>30</td>
      <td>London</td>
      <td>2022-11-20</td>
    </tr>
  </tbody>
</table>

<script>
  // Simple sort functionality
  document.querySelectorAll('.sortable-table th').forEach(header => {
    header.addEventListener('click', () => {
      const table = header.closest('table');
      const tbody = table.querySelector('tbody');
      const rows = Array.from(tbody.querySelectorAll('tr'));

      rows.sort((a, b) => {
        const aVal = a.children[header.cellIndex].textContent;
        const bVal = b.children[header.cellIndex].textContent;
        return aVal.localeCompare(bVal);
      });

      rows.forEach(row => tbody.appendChild(row));
    });
  });
</script>
```

## üìã Best Practices

### ‚úÖ DO:

```html
<!-- Use semantic table structure -->
<table>
  <caption>
    Employee Directory
  </caption>
  <thead>
    <tr>
      <th scope="col">Name</th>
      <th scope="col">Department</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">John Doe</th>
      <td>Engineering</td>
    </tr>
  </tbody>
</table>

<!-- Always provide captions -->
<table>
  <caption>
    Quarterly sales figures in thousands of dollars
  </caption>
  <!-- Table content -->
</table>

<!-- Use proper header associations -->
<table>
  <thead>
    <tr>
      <th id="name-header">Name</th>
      <th id="age-header">Age</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td headers="name-header">John</td>
      <td headers="age-header">25</td>
    </tr>
  </tbody>
</table>

<!-- Make tables responsive -->
<div class="table-container">
  <table>
    <!-- Table content -->
  </table>
</div>
```

### ‚ùå DON'T:

```html
<!-- Don't use tables for layout -->
<table>
  <tr>
    <td>Header</td>
    <td>Navigation</td>
  </tr>
  <tr>
    <td>Main content</td>
    <td>Sidebar</td>
  </tr>
</table>
<!-- Use CSS Grid or Flexbox instead -->

<!-- Don't skip thead/tbody -->
<table>
  <tr>
    <th>Name</th>
  </tr>
  <tr>
    <td>John</td>
  </tr>
</table>
<!-- Missing thead, tbody -->

<!-- Don't use th for styling -->
<table>
  <tr>
    <th style="font-weight: normal;">Not a header</th>
  </tr>
</table>
<!-- Use td instead -->

<!-- Don't forget alt text for data tables -->
<table>
  <!-- Missing caption and proper headers -->
</table>
```
