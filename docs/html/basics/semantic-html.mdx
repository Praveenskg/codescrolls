# üéØ Semantic HTML

Semantic HTML uses meaningful elements that clearly describe their purpose to
browsers, developers, and assistive technologies.

## üìñ Why Semantic HTML Matters?

Without semantic HTML:

```html
<div id="header">
  <div class="nav">
    <div class="nav-item">Home</div>
  </div>
</div>

<div class="main-content">
  <div class="post">
    <div class="title">Article Title</div>
    <div class="text">Article content...</div>
  </div>
</div>
```

With semantic HTML:

```html
<header>
  <nav>
    <a href="/">Home</a>
  </nav>
</header>

<main>
  <article>
    <h1>Article Title</h1>
    <p>Article content...</p>
  </article>
</main>
```

## üèóÔ∏è Document Structure Elements

### `<header>`

Represents introductory content or navigation.

```html
<!-- Site header -->
<header>
  <h1>Site Title</h1>
  <nav>
    <ul>
      <li><a href="/">Home</a></li>
      <li><a href="/about">About</a></li>
    </ul>
  </nav>
</header>

<!-- Article header -->
<article>
  <header>
    <h1>Article Title</h1>
    <p>By <a href="/author/john">John Doe</a></p>
    <time datetime="2025-01-15">January 15, 2025</time>
  </header>
</article>
```

### `<nav>`

Defines navigation links.

```html
<nav>
  <ul>
    <li><a href="/">Home</a></li>
    <li><a href="/products">Products</a></li>
    <li><a href="/about">About</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>

<!-- Secondary navigation -->
<nav aria-label="Product categories">
  <ul>
    <li><a href="/electronics">Electronics</a></li>
    <li><a href="/books">Books</a></li>
  </ul>
</nav>
```

### `<main>`

Represents the main content of the document.

```html
<body>
  <header>Site header</header>

  <main>
    <article>
      <h1>Main article</h1>
      <p>Article content...</p>
    </article>
  </main>

  <footer>Site footer</footer>
</body>
```

### `<section>`

Represents a standalone section of content.

```html
<main>
  <section id="introduction">
    <h2>Introduction</h2>
    <p>Welcome to our website...</p>
  </section>

  <section id="services">
    <h2>Our Services</h2>
    <ul>
      <li>Web Development</li>
      <li>Design</li>
      <li>Consulting</li>
    </ul>
  </section>

  <section id="contact">
    <h2>Contact Us</h2>
    <form>...</form>
  </section>
</main>
```

### `<article>`

Represents self-contained content that could be distributed independently.

```html
<!-- Blog post -->
<article>
  <header>
    <h1>Understanding React Hooks</h1>
    <p>Published by <a href="/authors/jane">Jane Doe</a></p>
    <time datetime="2025-01-10">January 10, 2025</time>
  </header>

  <p>React hooks are a powerful feature...</p>

  <footer>
    <p>Tags: <a href="/tags/react">React</a></p>
  </footer>
</article>

<!-- News article -->
<article>
  <h1>Breaking News</h1>
  <p>Important announcement...</p>
  <footer>¬© 2025 News Corp</footer>
</article>
```

### `<aside>`

Represents content indirectly related to the main content.

```html
<!-- Sidebar with related content -->
<aside>
  <h3>Related Articles</h3>
  <ul>
    <li><a href="/related1">Related Article 1</a></li>
    <li><a href="/related2">Related Article 2</a></li>
  </ul>
</aside>

<!-- Pull quote -->
<article>
  <p>Main content here...</p>

  <aside>
    <blockquote>"This is an important point worth highlighting."</blockquote>
  </aside>

  <p>More content...</p>
</article>
```

### `<footer>`

Represents footer content for its nearest sectioning content.

```html
<!-- Site footer -->
<footer>
  <p>&copy; 2025 Company Name</p>
  <nav>
    <a href="/privacy">Privacy Policy</a>
    <a href="/terms">Terms of Service</a>
  </nav>
</footer>

<!-- Article footer -->
<article>
  <h1>Article Title</h1>
  <p>Article content...</p>

  <footer>
    <p>Published: <time datetime="2025-01-01">January 1, 2025</time></p>
    <p>Author: John Doe</p>
  </footer>
</article>
```

## üìù Text Content Elements

### `<h1>` - `<h6>` Headings

Proper heading hierarchy is crucial for accessibility.

```html
<!-- Good hierarchy -->
<h1>Main Page Title</h1>

<section>
  <h2>Section Title</h2>

  <section>
    <h3>Subsection Title</h3>
    <h4>Sub-subsection</h4>
  </section>
</section>

<section>
  <h2>Another Section</h2>
</section>
```

### `<p>` Paragraphs

```html
<article>
  <h1>Article Title</h1>

  <p>This is the first paragraph of the article.</p>

  <p>This is the second paragraph with more detailed information.</p>
</article>
```

### `<blockquote>` Blockquotes

```html
<p>According to the documentation:</p>

<blockquote cite="https://developer.mozilla.org/">
  <p>
    The blockquote element represents content that is quoted from another
    source.
  </p>
  <footer>‚Äî MDN Web Docs</footer>
</blockquote>
```

### `<cite>` Citations

```html
<p>The book <cite>The Pragmatic Programmer</cite> is highly recommended.</p>

<blockquote>
  <p>
    Programs must be written for people to read, and only incidentally for
    machines to execute.
  </p>
</blockquote>
<p>‚Äî <cite>The Structure and Interpretation of Computer Programs</cite></p>
```

### `<time>` Time and Dates

```html
<article>
  <header>
    <h1>Conference Announcement</h1>
    <p>Published: <time datetime="2025-03-15">March 15, 2025</time></p>
  </header>

  <p>
    The conference will be held on
    <time datetime="2025-06-01T09:00">June 1, 2025 at 9:00 AM</time>.
  </p>
</article>
```

## üìä Data and Lists

### `<figure>` and `<figcaption>`

Groups media content with captions.

```html
<figure>
  <img src="chart.jpg" alt="Sales chart showing growth" />
  <figcaption>Quarterly sales growth from Q1 to Q4 2024</figcaption>
</figure>

<figure>
  <pre><code>console.log('Hello, World!');</code></pre>
  <figcaption>Your first JavaScript program</figcaption>
</figure>
```

### `<details>` and `<summary>`

Creates collapsible content.

```html
<details>
  <summary>What is HTML?</summary>
  <p>
    HTML (HyperText Markup Language) is the standard markup language for web
    pages.
  </p>
</details>

<details open>
  <summary>Advanced Options</summary>
  <form>
    <input type="checkbox" id="option1" />
    <label for="option1">Enable advanced features</label>
  </form>
</details>
```

### `<mark>` Highlighted Text

```html
<p>
  The most important point is <mark>semantic HTML improves accessibility</mark>.
</p>

<p>
  Search results: <mark>semantic</mark> HTML is better than generic
  <code>&lt;div&gt;</code> elements.
</p>
```

```html

<header>
  <nav>
    <h2>Site Navigation</h2>
    <ul>
      <li><a href="/">Home</a></li>

  </nav>

  <article>
```

```html

<nav>Main navigation</nav>
<main>Main content</main>
```

## üìã Best Practices

### ‚úÖ DO:

```html
<!-- Use semantic elements -->
<header>Site header</header>
<nav>Navigation</nav>
<main>Main content</main>
<section>Content sections</section>
<article>Articles</article>
<aside>Sidebars</aside>
<footer>Footers</footer>

<!-- Proper heading hierarchy -->
<h1>Main title</h1>
<section>
  <h2>Section title</h2>
  <section>
    <h3>Subsection</h3>
  </section>
</section>

<!-- Use figure for images with captions -->
<figure>
  <img src="diagram.jpg" alt="Process diagram" />
  <figcaption>Visual representation of the process</figcaption>
</figure>

<!-- Use time for dates -->
<p>Published: <time datetime="2025-01-01">January 1, 2025</time></p>
```

### ‚ùå DON'T:

```html
<!-- Don't use div for everything -->
<div id="header">Header</div>
<!-- Use <header> -->
<div class="nav">Navigation</div>
<!-- Use <nav> -->
<div class="main">Content</div>
<!-- Use <main> -->

<!-- Don't skip heading levels -->
<h1>Title</h1>
<h3>Skip h2</h3>
<!-- Poor hierarchy -->

<!-- Don't use blockquote for indentation -->
<blockquote>Indented text</blockquote>
<!-- Wrong usage -->

<!-- Don't use header/footer inside article improperly -->
<article>
  <header>Article header</header>
  <header>Another header</header>
  <!-- Only one per article -->
</article>
```

## üîç Semantic vs Generic Elements

| Generic  | Semantic    | Use Case             |
| -------- | ----------- | -------------------- |
| `<div>`  | `<header>`  | Page/article header  |
| `<div>`  | `<nav>`     | Navigation menus     |
| `<div>`  | `<main>`    | Main content         |
| `<div>`  | `<section>` | Content sections     |
| `<div>`  | `<article>` | Independent content  |
| `<div>`  | `<aside>`   | Sidebar content      |
| `<div>`  | `<footer>`  | Footer content       |
| `<span>` | `<time>`    | Dates and times      |
| `<div>`  | `<figure>`  | Images with captions |
| `<span>` | `<mark>`    | Highlighted text     |
