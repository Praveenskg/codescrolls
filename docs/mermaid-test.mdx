# Mermaid Diagrams in CodeScrolls

This page demonstrates the Mermaid diagram support in CodeScrolls documentation.

## Flowchart

```mermaid
graph TD
    A[Start] --> B{Is it working?}
    B -->|Yes| C[Great!]
    B -->|No| D[Check config]
    D --> E[Fix issues]
    E --> B
    C --> F[End]
```

## Sequence Diagram

```mermaid
sequenceDiagram
    participant User
    participant Browser
    participant Server
    participant Database

    User->>Browser: Click button
    Browser->>Server: HTTP Request
    Server->>Database: Query data
    Database-->>Server: Return results
    Server-->>Browser: HTTP Response
    Browser-->>User: Update UI
```

## Class Diagram

```mermaid
classDiagram
    class User {
        +String name
        +String email
        +login()
        +logout()
    }

    class Post {
        +String title
        +String content
        +Date createdAt
        +publish()
        +edit()
    }

    class Comment {
        +String content
        +Date createdAt
        +reply()
    }

    User ||--o{ Post : creates
    Post ||--o{ Comment : has
    User ||--o{ Comment : writes
```

## State Diagram

```mermaid
stateDiagram-v2
    [*] --> Idle
    Idle --> Processing : start
    Processing --> Success : complete
    Processing --> Error : fail
    Error --> Idle : retry
    Success --> [*]
```

## Gantt Chart

```mermaid
gantt

    dateFormat  YYYY-MM-DD
    section Planning
    Define requirements    :done, req, 2024-01-01, 2024-01-07
    Create wireframes      :done, wire, after req, 2024-01-05

    section Development
    Setup project          :done, setup, 2024-01-08, 2024-01-10
    Implement features     :active, impl, after setup, 2024-02-15
    Write tests            :test, after impl, 2024-02-10

    section Deployment
    Deploy to staging      :staging, after test, 2024-02-20
    Deploy to production   :prod, after staging, 2024-02-25
```
