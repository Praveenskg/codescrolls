## üöÄ What's New in React 19 {#whats-new-in-react-19}

React 19 brings several groundbreaking features:

- **React Compiler**: Automatic optimization without manual memoization
- **Form Actions**: Simplified form handling with server actions
- **New Hooks**: `useActionState`, `useFormStatus`, `useOptimistic`, `use()`
- **Server Components**: Built-in support for server-side rendering
- **Enhanced SSR**: Improved streaming and hydration
- **Better TypeScript Support**: Improved type inference and error messages
- **Automatic Batching**: More efficient state updates

## ‚ö° React Compiler {#react-compiler}

The React Compiler automatically optimizes your components by memoizing values
and callbacks, eliminating the need for manual `useMemo`, `useCallback`, and
`React.memo` in most cases.

### Before React Compiler

```tsx
import { useState, useMemo, useCallback } from 'react';

function ProductList({ products }) {
  const [filter, setFilter] = useState('');

  // Manual memoization required
  const filteredProducts = useMemo(() => {
    return products.filter(p => p.name.includes(filter));
  }, [products, filter]);

  const handleClick = useCallback(id => {
    console.log('Clicked:', id);
  }, []);

  return (
    <div>
      <input value={filter} onChange={e => setFilter(e.target.value)} />
      {filteredProducts.map(product => (
        <Product key={product.id} product={product} onClick={handleClick} />
      ))}
    </div>
  );
}
```

### With React Compiler

```tsx
import { useState } from 'react';

function ProductList({ products }) {
  const [filter, setFilter] = useState('');

  // React Compiler automatically memoizes this
  const filteredProducts = products.filter(p => p.name.includes(filter));

  // React Compiler automatically memoizes this callback
  const handleClick = id => {
    console.log('Clicked:', id);
  };

  return (
    <div>
      <input value={filter} onChange={e => setFilter(e.target.value)} />
      {filteredProducts.map(product => (
        <Product key={product.id} product={product} onClick={handleClick} />
      ))}
    </div>
  );
}
```

The React Compiler analyzes your code and automatically applies optimizations,
making your code cleaner and more performant without manual intervention.

## üìù Form Actions & New Hooks {#form-actions--new-hooks}

React 19 introduces a powerful new way to handle forms with server actions and
several new hooks that simplify form state management.

### Form Actions

Form Actions allow you to handle form submissions directly with server
functions, reducing the need for client-side state management.

```tsx
// Server Action (Next.js App Router example)
async function createUser(formData: FormData) {
  'use server';

  const name = formData.get('name') as string;
  const email = formData.get('email') as string;

  // Validate and save to database
  await db.users.create({ name, email });

  return { success: true, message: 'User created successfully' };
}

// Client Component
function UserForm() {
  return (
    <form action={createUser}>
      <input name="name" type="text" required />
      <input name="email" type="email" required />
      <button type="submit">Create User</button>
    </form>
  );
}
```

### useActionState {#useactionstate}

`useActionState` (formerly `useFormState`) manages form state and handles action
results.

```tsx
import { useActionState } from 'react';

async function updateProfile(prevState, formData) {
  const name = formData.get('name');

  try {
    await updateUserProfile(name);
    return { success: true, message: 'Profile updated!' };
  } catch (error) {
    return { success: false, message: error.message };
  }
}

function ProfileForm() {
  const [state, formAction, isPending] = useActionState(updateProfile, null);

  return (
    <form action={formAction}>
      <input name="name" type="text" required />
      <button type="submit" disabled={isPending}>
        {isPending ? 'Updating...' : 'Update Profile'}
      </button>
      {state?.message && (
        <p style={{ color: state.success ? 'green' : 'red' }}>
          {state.message}
        </p>
      )}
    </form>
  );
}
```

### useFormStatus {#useformstatus}

`useFormStatus` provides form submission status within form components, perfect
for showing loading states.

```tsx
import { useFormStatus } from 'react';

function SubmitButton() {
  const { pending, data, method, action } = useFormStatus();

  return (
    <button type="submit" disabled={pending}>
      {pending ? 'Submitting...' : 'Submit'}
    </button>
  );
}

function ContactForm() {
  async function submitForm(formData) {
    'use server';
    // Handle form submission
  }

  return (
    <form action={submitForm}>
      <input name="email" type="email" required />
      <input name="message" type="text" required />
      <SubmitButton />
    </form>
  );
}
```

### useOptimistic {#useoptimistic}

`useOptimistic` enables optimistic UI updates, showing immediate feedback before
server confirmation.

```tsx
import { useOptimistic, useActionState } from 'react';

async function addComment(formData) {
  'use server';
  const comment = formData.get('comment');
  // Simulate API call
  await new Promise(resolve => setTimeout(resolve, 1000));
  return { id: Date.now(), text: comment };
}

function CommentList({ comments }) {
  const [optimisticComments, addOptimisticComment] = useOptimistic(
    comments,
    (state, newComment) => [...state, { ...newComment, pending: true }],
  );

  async function formAction(formData) {
    addOptimisticComment({ id: 'temp', text: formData.get('comment') });
    return await addComment(formData);
  }

  const [state, action] = useActionState(formAction, null);

  return (
    <div>
      <form action={action}>
        <input name="comment" type="text" required />
        <button type="submit">Add Comment</button>
      </form>
      <ul>
        {optimisticComments.map(comment => (
          <li key={comment.id}>
            {comment.text}
            {comment.pending && <span> (saving...)</span>}
          </li>
        ))}
      </ul>
    </div>
  );
}
```

## üé£ The `use()` Hook {#the-use-hook}

The `use()` hook allows you to read the value of a Promise or Context directly
in components, including conditional usage.

```tsx
import { use, Suspense } from 'react';

function UserProfile({ userPromise }) {
  // Can be used conditionally, unlike hooks before
  const user = use(userPromise);

  return (
    <div>
      <h1>{user.name}</h1>
      <p>{user.email}</p>
    </div>
  );
}

function App() {
  const userPromise = fetchUser();

  return (
    <Suspense fallback={<div>Loading...</div>}>
      <UserProfile userPromise={userPromise} />
    </Suspense>
  );
}
```

### Using `use()` with Context

```tsx
import { createContext, use } from 'react';

const ThemeContext = createContext(null);

function ThemedButton() {
  // Can be used conditionally
  const theme = use(ThemeContext);

  return <button style={{ background: theme.primary }}>Themed Button</button>;
}
```

## üñ•Ô∏è Server Components {#server-components}

React 19 includes built-in support for Server Components, allowing you to fetch
data and render on the server.

```tsx
// Server Component (Next.js App Router)
async function ProductPage({ id }) {
  // This runs on the server
  const product = await fetchProduct(id);

  return (
    <div>
      <h1>{product.name}</h1>
      <p>{product.description}</p>
      <ProductPrice price={product.price} />
    </div>
  );
}

// Client Component (marked with 'use client')
('use client');

function AddToCartButton({ productId }) {
  const [isAdding, setIsAdding] = useState(false);

  async function handleAddToCart() {
    setIsAdding(true);
    await addToCart(productId);
    setIsAdding(false);
  }

  return (
    <button onClick={handleAddToCart} disabled={isAdding}>
      {isAdding ? 'Adding...' : 'Add to Cart'}
    </button>
  );
}
```

## üÜï React 19.2 Features {#react-192-features}

React 19.2 introduces additional features that enhance the developer experience.

### Activity Component {#activity-component}

The `<Activity />` component provides a standardized way to show loading and
activity states.

```tsx
import { Activity } from 'react';

function DataFetcher() {
  const [data, setData] = useState(null);
  const [isLoading, setIsLoading] = useState(true);

  useEffect(() => {
    fetchData().then(result => {
      setData(result);
      setIsLoading(false);
    });
  }, []);

  if (isLoading) {
    return <Activity />;
  }

  return <div>{/* Render data */}</div>;
}
```

### useEffectEvent Hook {#useeffectevent-hook}

`useEffectEvent` allows you to define event handlers that don't need to be in
the dependency array.

```tsx
import { useEffect, useEffectEvent, useState } from 'react';

function ChatRoom({ roomId }) {
  const [message, setMessage] = useState('');

  // This function doesn't need to be in dependencies
  const onConnected = useEffectEvent(() => {
    showNotification('Connected to ' + roomId);
  });

  useEffect(() => {
    const connection = createConnection(roomId);
    connection.on('connected', onConnected);
    connection.connect();

    return () => connection.disconnect();
  }, [roomId]); // onConnected is not needed here

  return (
    <div>
      <input value={message} onChange={e => setMessage(e.target.value)} />
    </div>
  );
}
```

### cacheSignal API {#cachesignal-api}

`cacheSignal` provides a way to cache signals for better performance in
concurrent rendering.

```tsx
import { cacheSignal } from 'react';

function ExpensiveComponent({ data }) {
  const cachedData = cacheSignal(() => {
    // Expensive computation
    return processData(data);
  });

  return <div>{cachedData.value}</div>;
}
```

## ‚ö° Performance Improvements {#performance-improvements}

### Automatic Batching

React 19 automatically batches more state updates, including updates from
promises, timeouts, and native event handlers.

```tsx
function App() {
  const [count, setCount] = useState(0);
  const [flag, setFlag] = useState(false);

  function handleClick() {
    // These are automatically batched in React 19
    setCount(c => c + 1);
    setFlag(f => !f);

    // Even async updates are batched
    setTimeout(() => {
      setCount(c => c + 1);
      setFlag(f => !f);
    }, 1000);
  }

  return <button onClick={handleClick}>Click me</button>;
}
```

### Enhanced SSR

React 19 improves Server-Side Rendering with better streaming, selective
hydration, and reduced JavaScript bundle size.

```tsx
// Server Component with streaming
async function BlogPost({ slug }) {
  const post = await fetchPost(slug);

  return (
    <article>
      <h1>{post.title}</h1>
      <Suspense fallback={<div>Loading comments...</div>}>
        <Comments postId={post.id} />
      </Suspense>
    </article>
  );
}
```

// Better inference for refs
function MyComponent() {

## üîÑ Migration Guide {#migration-guide}

### Upgrading to React 19

1. **Update Dependencies**

```bash
npm install react@19 react-dom@19
```

2. **Update TypeScript Types** (if using TypeScript)

```bash
npm install --save-dev @types/react@19 @types/react-dom@19
```

3. **Review Breaking Changes**
   - Some deprecated APIs have been removed
   - Check your codebase for warnings

4. **Enable React Compiler** (optional)

```json
{
  "compilerOptions": {
    "plugins": [
      {
        "name": "babel-plugin-react-compiler"
      }
    ]
  }
}
```

### Common Migration Patterns

#### Replacing useFormState with useActionState

```tsx
// Before (React 18)
import { useFormState } from 'react-dom';

// After (React 19)
import { useActionState } from 'react';
```

#### Removing Manual Memoization

```tsx
// Before
const memoizedValue = useMemo(() => computeExpensiveValue(a, b), [a, b]);
const memoizedCallback = useCallback(() => doSomething(a), [a]);

// After (with React Compiler)
const value = computeExpensiveValue(a, b);
const callback = () => doSomething(a);
```

## ‚úÖ Best Practices {#best-practices}

### 1. Use Form Actions for Form Handling

```tsx
// ‚úÖ Good: Use form actions
<form action={serverAction}>
  <input name="email" />
  <button type="submit">Submit</button>
</form>;

// ‚ùå Avoid: Manual form state management when not needed
const [email, setEmail] = useState('');
const [isSubmitting, setIsSubmitting] = useState(false);
```

### 2. Leverage React Compiler

```tsx
// ‚úÖ Good: Let React Compiler handle optimization
function Component({ items }) {
  const filtered = items.filter(item => item.active);
  return <List items={filtered} />;
}

// ‚ùå Avoid: Premature optimization without compiler
const filtered = useMemo(() => items.filter(...), [items]);
```

### 3. Use Server Components When Possible

```tsx
// ‚úÖ Good: Fetch data in Server Component
async function Page() {
  const data = await fetchData();
  return <Content data={data} />;
}

// ‚ùå Avoid: Fetching in Client Component when not needed
('use client');
function Page() {
  const [data, setData] = useState(null);
  useEffect(() => {
    fetchData().then(setData);
  }, []);
  // ...
}
```

### 4. Use useOptimistic for Better UX

```tsx
// ‚úÖ Good: Optimistic updates
const [items, addOptimistic] = useOptimistic(currentItems, ...);

// ‚ùå Avoid: Waiting for server response before UI update
const handleAdd = async () => {
  setIsLoading(true);
  const result = await addItem();
  setItems([...items, result]);
  setIsLoading(false);
};
```

### 5. Proper Error Handling

```tsx
// ‚úÖ Good: Handle errors in actions
async function action(formData) {
  'use server';
  try {
    await saveData(formData);
    return { success: true };
  } catch (error) {
    return { success: false, error: error.message };
  }
}
```

## üéØ Conclusion

React 19 and 19.2 bring significant improvements to the React ecosystem:

- **Simplified Development**: Form Actions and new hooks reduce boilerplate
- **Better Performance**: React Compiler and automatic batching optimize your
  apps
- **Enhanced DX**: Better TypeScript support and error messages
- **Modern Patterns**: Server Components and improved SSR capabilities

### Key Takeaways

1. **React Compiler** eliminates the need for manual memoization in most cases
2. **Form Actions** simplify form handling with server-side logic
3. **New Hooks** (`useActionState`, `useFormStatus`, `useOptimistic`, `use()`)
   provide powerful patterns
4. **Server Components** enable better performance and SEO
5. **React 19.2** adds polish with `useEffectEvent` and `<Activity />`

Start migrating your React applications to take advantage of these powerful new
features. The React team has made significant improvements that will make your
code cleaner, faster, and more maintainable.
